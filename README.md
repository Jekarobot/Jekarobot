### 🚀 **github-repo-manager** — Автоматизированный менеджер репозиториев

**Назначение:**  
Проект предназначен для автоматизации управления списком репозиториев в вашем GitHub-профиле. Он генерирует динамический файл `PROJECTS.md` с актуальной информацией о ваших проектах, используя данные из GitHub API и интеллектуальные сводки на основе ИИ.

**Ключевые технологии и стек:**  
- **Язык:** TypeScript (Node.js)  
- **Инфраструктура:** Docker, Docker Compose  
- **Веб-сервер:** Express.js (в `src/server/`)  
- **Внешние API:** GitHub REST API, DeepSeek AI (для генерации описаний)  
- **Инструменты:** CLI-интерфейс, конфигурация через JSON (`repos.config.example.json`)

**Архитектура и особенности реализации:**  
Проект построен по модульной архитектуре с разделением на CLI, сервер и сервисы.  
- **CLI (`src/cli/`)** — точка входа для запуска задач (например, обновление списка репозиториев).  
- **Сервер (`src/server/`)** — предоставляет REST API для ручного запуска обновлений с защитой через `push-confirm` middleware.  
- **Сервисы (`src/services/`)** — отвечают за взаимодействие с GitHub, генерацию сводок (через DeepSeek), обновление README и санитизацию данных.  
- **Конфигурация** — гибкая настройка через `repos.config.json` (исключение репозиториев, настройки для каждого проекта).

**Основные возможности:**  
- 🔄 **Автоматическое обновление** списка репозиториев в `PROJECTS.md` с описанием, звёздами и форками.  
- 🤖 **Генерация сводок** с помощью DeepSeek AI для каждого проекта (краткое описание на основе README).  
- 📦 **Docker-поддержка** — запуск в контейнере с помощью `docker-compose.yml` для лёгкого деплоя.  
- 🛡️ **Безопасность** — middleware `push-confirm` для подтверждения обновлений через API.  
- 🧹 **Санитизация данных** — фильтрация и очистка информации перед записью в файлы.

[🔗 GitHub Repo](https://github.com/Jekarobot/github-repo-manager)

### 🔔 **theia-extension-notifications** — Расширение уведомлений для Eclipse Theia

**Назначение проекта:** Расширение для Eclipse Theia, реализующее гибкую систему уведомлений с поддержкой реального времени. Проект позволяет интегрировать push-уведомления в IDE на базе Theia, обеспечивая разработчикам мгновенное получение информации о событиях в проекте, CI/CD пайплайнах и командных действиях.

**Ключевые технологии и стек:** 🛠️ TypeScript, Node.js, React (для UI-компонентов Theia), Docker, Yarn Workspaces. Архитектура построена на трехслойной модели: browser (фронтенд), common (общие интерфейсы) и node (серверная часть), что обеспечивает модульность и масштабируемость.

**Архитектура и особенности реализации:** 🏗️ Проект использует монорепозиторий с Yarn Workspaces, разделенный на два основных пакета: `theia-app` (конфигурация приложения Theia с Docker-контейнеризацией) и `theia-notification-extension` (само расширение). Взаимодействие между слоями реализовано через JSON-RPC протокол, что позволяет обрабатывать уведомления как в браузерном, так и в серверном окружении. Dockerfile в `theia-app` обеспечивает легкое развертывание в облачных средах.

**Основные возможности:** ✨
- Отображение уведомлений в реальном времени с приоритетами (info, warning, error)
- Кастомизация внешнего вида через тему Theia
- Поддержка группировки и фильтрации уведомлений
- REST API для внешних сервисов (CI/CD, мониторинг)
- Полная документация в `docs/` (архитектура, бэкенд, фронтенд, Docker)

[🔗 GitHub Repo](https://github.com/Jekarobot/theia-extension-notifications)

### 🔥 **myheat-onboard** — Онбординг для системы умного отопления

**Назначение проекта**  
Проект представляет собой одностраничное веб-приложение для онбординга пользователей системы «Мой Теплый Дом». Решение позволяет быстро познакомить новых клиентов с функционалом платформы, настроить первичные параметры отопления и визуализировать данные о теплопотреблении. Основная цель — упростить первый опыт взаимодействия с системой и снизить нагрузку на службу поддержки.

**Ключевые технологии и стек**  
- **Frontend**: HTML5, CSS3 (адаптивная верстка), Vanilla JavaScript (ES6+)  
- **Backend**: PHP 8.1 (API-эндпоинты)  
- **Инфраструктура**: Docker, Docker Compose (контейнеризация)  
- **Графика**: SVG-логотип, PNG/JPG-изображения для промо-материалов  

**Архитектура и особенности реализации**  
Приложение построено по принципу SPA (Single Page Application) с минимальным серверным API. Фронтенд полностью статический (index.html + index.js + index.css), все динамические данные загружаются через PHP-эндпоинты в папке `api/`. Контейнеризация через Docker обеспечивает изолированное окружение и простой деплой. Особое внимание уделено обработке потенциальных проблем (документировано в `POTENTIAL_PROBLEMS.md`) и четкому трекингу задач (`TASKS.md`).

**Основные возможности**  
- 🚀 Пошаговый онбординг с визуальными подсказками  
- 📊 Интерактивная панель с базовыми метриками отопления  
- 🎨 Адаптивный дизайн для мобильных устройств  
- 🔌 Готовое Docker-окружение для локальной разработки  
- 📁 Модульная структура с разделением на статику и API  

[🔗 GitHub Repo](https://github.com/Jekarobot/myheat-onboard)

### 🚂 **railway-booking** — Сервис для покупки железнодорожных билетов

**Назначение проекта**  
Railway Booking — это современное веб-приложение для поиска и бронирования железнодорожных билетов. Проект предоставляет пользователям удобный интерфейс для выбора маршрутов, просмотра детальной информации о поездах и оформления заказов. Решение ориентировано на создание бесшовного пользовательского опыта при планировании поездок.

**Ключевые технологии и стек**  
- **React 18** с TypeScript для типизированной разработки  
- **Vite** для быстрой сборки и горячей перезагрузки модулей  
- **ESLint** с конфигурацией для поддержания качества кода  
- **Feature-Sliced Design (FSD)** — архитектурная методология для масштабируемости  

**Архитектура и особенности реализации**  
Проект построен на принципах **Feature-Sliced Design**, что обеспечивает четкое разделение ответственности между слоями: `entities`, `features`, `widgets`, `pages` и `shared`. Используется **контекстное управление состоянием** через провайдеры (`SearchProvider`, `OrderBuildProvider`, `PopupProvider`, `TrainDetailsProvider`), что позволяет изолировать логику различных модулей. В проекте реализована **кастомная система компонентов** (Calendar, Slider, Loader, Popup) для гибкой настройки интерфейса.

**Основные возможности**  
- 🔍 **Интеллектуальный поиск билетов** с фильтрацией по дате, времени и типу поезда  
- 🗓 **Кастомный календарь** для выбора дат с визуализацией доступных рейсов  
- 🚄 **Детальная информация о поездах** с отображением схемы вагонов и цен  
- 🛒 **Полный цикл оформления заказа** — от выбора места до оплаты  
- 📱 **Адаптивный дизайн** для комфортного использования на любых устройствах  
- 🎨 **Модульные виджеты** (AboutSection, FeedbackSection, HowItWorksSection) для расширения функциональности  

[🔗 GitHub Repo](https://github.com/Jekarobot/railway-booking)
