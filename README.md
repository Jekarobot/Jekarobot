### 🚀 GitHub Repo Manager

**GitHub Repo Manager** — это инструмент для автоматизации управления репозиториями и генерации динамического профиля GitHub. Проект позволяет синхронизировать список проектов в `PROJECTS.md` с реальными данными из GitHub API, а также автоматически обновлять `README.md` профиля с краткой сводкой по каждому репозиторию. Решение подходит для разработчиков, желающих поддерживать актуальное портфолио без ручного редактирования.

#### 🛠️ Ключевые технологии и стек

- **TypeScript** — основной язык разработки, обеспечивающий типовую безопасность.
- **Node.js** — среда выполнения для CLI и серверной части.
- **Express.js** — веб-сервер для обработки запросов и вебхуков.
- **GitHub API (REST)** — получение данных о репозиториях, коммитах и релизах.
- **DeepSeek API** — генерация кратких описаний проектов с помощью ИИ.
- **Docker** — контейнеризация для простого развертывания.

#### 🏗️ Архитектура и особенности реализации

Проект разделен на три модуля: **CLI** (командная строка для ручного запуска), **сервер** (автоматическая обработка вебхуков от GitHub) и **сервисы** (бизнес-логика). Конфигурация хранится в `repos.config.json`, а логирование ведется через централизованный `logger.ts`. Особенность — использование `push-confirm.ts` middleware для подтверждения пушей через вебхуки, что обеспечивает безопасность при автоматическом обновлении профиля.

#### ✨ Основные возможности

- **Автоматическая синхронизация** — обновление `PROJECTS.md` и `README.md` при пуше в репозиторий.
- **Генерация сводок** — создание кратких описаний проектов с помощью DeepSeek AI.
- **Поддержка Docker** — запуск в контейнере с помощью `docker-compose.yml`.
- **Гибкая конфигурация** — настройка через JSON-файл с примерами (`repos.config.example.json`).
- **Безопасность** — проверка подлинности вебхуков через `push-confirm.ts`.

### 📊 bjs-diplom — Демонстрационное веб-приложение для управления финансами

#### 🎯 Назначение проекта

Проект **bjs-diplom** представляет собой демонстрационное веб-приложение, разработанное в рамках дипломной работы. Оно имитирует функциональность личного кабинета финансового сервиса, позволяя пользователям просматривать курсы валют, управлять избранными акциями и контролировать баланс счета. Приложение демонстрирует навыки работы с серверной частью на Node.js, клиент-серверным взаимодействием и современными CSS-фреймворками.

#### 🛠️ Ключевые технологии и стек

- **Backend:** Node.js, Express.js (маршрутизация, обработка запросов)
- **Frontend:** Чистый JavaScript (ES6+), HTML5, CSS3
- **UI-фреймворк:** Semantic UI (стилизация компонентов)
- **Данные:** JSON-файл (`db.json`) для хранения пользовательских данных и курсов
- **Сборка:** package.json для управления зависимостями

#### 🏗️ Архитектура и особенности реализации

Приложение построено по классической клиент-серверной архитектуре. Серверная часть (файлы в папке `routes/`) обрабатывает запросы к API, взаимодействуя с файлом `db.json`. Клиентская часть (папка `public/`) включает отдельные страницы для входа (`login.html`) и главной страницы (`home.html`), каждая со своим JavaScript-файлом. API-коннекторы (`ApiConnector.js`, `FavoritesWidget.js` и др.) обеспечивают модульное взаимодействие с сервером. Semantic UI используется для быстрой и адаптивной верстки интерфейса.

#### ✨ Основные возможности

- **Авторизация пользователей:** Форма входа с проверкой учетных данных
- **Просмотр курсов валют и акций:** Динамическое обновление данных с сервера
- **Управление избранными акциями:** Добавление/удаление акций в избранное
- **Мониторинг баланса:** Отображение текущего состояния счета и истории операций
- **Выход из системы:** Безопасное завершение сессии

### 🏦 BHJ Diploma — Система управления банковскими счетами и транзакциями

**Назначение проекта:**  
BHJ Diploma — это полнофункциональное веб-приложение для управления банковскими счетами и транзакциями, разработанное в рамках дипломного проекта. Система позволяет пользователям регистрироваться, создавать счета, выполнять переводы и отслеживать историю операций в реальном времени.

**Ключевые технологии и стек:**  
Проект построен на чистом **JavaScript (ES6+)** с использованием **Node.js** и **Express** для серверной части. Фронтенд реализован на нативном JS с применением **AdminLTE** для UI-компонентов, а данные хранятся в **JSON-файле** (db.json) с REST API-маршрутами.

**Архитектура и особенности реализации:**  
Архитектура следует принципам **MVC** с разделением на маршруты (routes/account.js, transaction.js, user.js), API-слой (api/) и UI-компоненты (ui/). Особое внимание уделено асинхронной обработке форм, модальным окнам и динамическому обновлению боковой панели. Все запросы к серверу выполняются через **fetch API** с обработкой ошибок.

**Основные возможности:**  
- 🔐 **Регистрация и аутентификация** пользователей  
- 💳 **Создание и управление счетами** (sidebar-accounts)  
- 💸 **Выполнение транзакций** с валидацией  
- 📊 **Отображение истории операций** с фильтрацией  
- 🖥️ **Интерактивная боковая панель** с переключением разделов  
- 📝 **Асинхронные формы** с обратной связью через модальные окна

🔗 [Перейти к проекту](https://github.com/Jekarobot/bhj-diploma)

### 🎯 Проект "DOM" — Интерактивная игра на чистом JavaScript

#### 📌 Назначение
Проект представляет собой браузерную мини-игру, в которой пользователь должен кликать по появляющемуся гоблину, нанося ему удары молотком. Игра реализована с использованием нативного DOM API и предназначена для отработки навыков манипуляции элементами страницы, обработки событий и анимации.

#### 🛠️ Ключевые технологии и стек
- **JavaScript (ES6+)** — вся логика игры написана на чистом JS без фреймворков
- **Webpack 5** — сборка проекта с разделением на dev, prod и общую конфигурацию
- **ESLint** — статический анализ кода с современными правилами
- **CSS3** — стилизация интерфейса и анимации появления/исчезновения гоблина
- **Yarn** — менеджер зависимостей

#### 🏗️ Архитектура и особенности реализации
Проект имеет модульную структуру: основной файл `index.js` инициализирует приложение, `app.js` содержит игровую логику, а все стили вынесены в отдельный CSS-файл. Webpack настроен на три режима: общий (`webpack.common.js`), разработки (`webpack.dev.js`) и продакшена (`webpack.prod.js`), что обеспечивает гибкость сборки. Тесты расположены в отдельной директории `tests` для удобства поддержки.

#### ✨ Основные возможности
- 🎮 Интерактивный геймплей с кликами по динамически перемещающемуся гоблину
- 🖼️ Визуальные эффекты с использованием SVG-логотипа и спрайтов (молоток, гоблин)
- 📱 Адаптивный интерфейс, стилизованный под единый дизайн
- ⚡ Быстрая загрузка благодаря оптимизированной сборке Webpack
- 🧪 Подготовленная инфраструктура для модульного тестирования

### 🚆 Railway Booking — Сервис для поиска и бронирования железнодорожных билетов

#### 🎯 Назначение проекта

Railway Booking — это клиентское веб-приложение для поиска, сравнения и бронирования железнодорожных билетов. Проект предоставляет пользователям удобный интерфейс для выбора маршрутов, просмотра деталей поездов и оформления заказа. Решение ориентировано на имитацию реального сервиса бронирования с полным циклом взаимодействия.

#### 🛠️ Ключевые технологии и стек

- **React 18** + **TypeScript** — для типизированной и производительной разработки UI
- **Vite** — быстрая сборка и горячая перезагрузка модулей
- **Feature-Sliced Design (FSD)** — архитектурная методология для масштабируемости и переиспользования кода
- **ESLint** — статический анализ кода с конфигурацией для React и TypeScript
- **CSS-модули** — изолированные стили для каждого компонента

#### 🏗️ Архитектура и особенности реализации

Проект строго следует методологии **Feature-Sliced Design**, что обеспечивает чёткое разделение ответственности между слоями: `app`, `pages`, `features`, `entities`, `widgets`, `shared` и `providers`. Контексты (`SearchProvider`, `OrderBuildProvider`, `PopupProvider`, `TrainDetailsProvider`) управляют глобальным состоянием, а моковые данные в `shared/mockData` позволяют тестировать функциональность без бэкенда. Кастомные хуки в `shared/hooks` инкапсулируют повторяющуюся логику, такую как работа с API и управление формами.

#### ✨ Основные возможности

- **🔍 Поиск билетов** — фильтрация по дате, направлению и количеству пассажиров
- **📋 Список поездов** — отображение маршрутов с деталями (время, цена, тип вагона)
- **🎨 Интерактивный календарь** — выбор даты с подсветкой доступных рейсов
- **🛒 Оформление заказа** — пошаговая форма с вводом данных пассажиров и выбором мест
- **📱 Адаптивный дизайн** — корректное отображение на мобильных устройствах и десктопах
- **⚡ Лоадеры и попапы** — плавная обратная связь при загрузке и подтверждении действий

🔗 [Перейти к проекту](https://github.com/Jekarobot/railway-booking)
