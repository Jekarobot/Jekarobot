### 📦 GitHub Repo Manager

**Назначение:** Инструмент для автоматизации управления GitHub-репозиториями и обновления профиля README. Проект позволяет генерировать структурированные списки проектов, создавать саммари на основе AI-анализа и автоматически синхронизировать данные между репозиториями и профилем пользователя.

**Технологии и стек:** TypeScript, Node.js, DeepSeek API (AI-генерация саммари), HTML/CSS/JavaScript (веб-интерфейс), Docker, docker-compose.

**Архитектура и особенности:** Разделена на три модуля — CLI, сервер с REST API и сервисы для работы с GitHub, DeepSeek и файловой системой. Модульная структура с чётким разделением ответственности: каждый сервис отвечает за свою задачу (работа с репозиториями, генерация README, санитизация данных). Конфигурация гибко настраивается через JSON-файлы.

**Основные возможности:**
- 🔄 Автоматическая синхронизация репозиториев с профилем README
- 🤖 AI-генерация саммари проектов с помощью DeepSeek
- 📝 Генерация структурированных списков проектов в формате Markdown
- 🚀 Push-уведомления с подтверждением изменений
- 🧹 Санитизация данных для безопасной работы с пользовательским вводом
- 🐳 Docker-поддержка для простого развертывания
- ⚙️ Гибкая конфигурация через JSON-файлы и переменные окружения

[🔗 Ссылка на репозиторий](https://github.com/Jekarobot/github-repo-manager)

### 🏦 BHJ-Diploma — Система управления банковскими счетами

**Назначение:** Веб-приложение для управления банковскими счетами и транзакциями. Система позволяет пользователям создавать счета, выполнять переводы средств и отслеживать историю операций через интуитивно понятный интерфейс. Дипломный проект, демонстрирующий навыки построения полноценного клиент-серверного приложения.

**Технологии и стек:** JavaScript (ES6+), HTML5, CSS3, AdminLTE (шаблон интерфейса), Node.js, Express.js, JSON-файл (`db.json`) в качестве базы данных.

**Архитектура и особенности:** Одностраничное приложение (SPA) с серверной частью на Express. Фронтенд организован в виде модулей: `App.js` — главный контроллер, `api/` — слой взаимодействия с сервером, `ui/` — компоненты интерфейса. Серверная часть включает маршруты для работы с пользователями (`routes/user.js`), счетами (`routes/account.js`) и транзакциями (`routes/transaction.js`). Документация API и руководства по функциям хранятся в папке `md/`.

**Основные возможности:**
- 🔐 Регистрация и аутентификация пользователей с хранением сессий
- 💳 Создание и управление счетами с отображением баланса
- 💸 Выполнение транзакций между счетами с валидацией данных
- 📊 Просмотр истории операций с фильтрацией и сортировкой
- 🎨 Адаптивный интерфейс на основе AdminLTE с боковым меню и модальными окнами
- 📝 Асинхронные формы для ввода данных без перезагрузки страницы

[🔗 Ссылка на репозиторий](https://github.com/Jekarobot/bhj-diploma)

### 🔥 MyHeat-Onboard

**Назначение:** Легковесное веб-приложение для управления и мониторинга системы отопления "MyHeat". Проект предоставляет удобный интерфейс для быстрого запуска и настройки оборудования, а также отображает ключевые параметры работы системы в реальном времени.

**Технологии и стек:** HTML5, CSS3, JavaScript (Vanilla JS), PHP (API-слой), Docker, Docker Compose, SVG (логотип), JPG/PNG (промо-изображения).

**Архитектура и особенности:** Клиент-серверная архитектура. Frontend — одностраничное приложение (SPA) с динамическим обновлением данных. Backend — REST API для обработки запросов и взаимодействия с системой отопления. Контейнеризация обеспечивает изолированное окружение и простой деплой. Минималистичный дизайн с акцентом на читаемость данных, асинхронная загрузка информации через Fetch API, поддержка адаптивной верстки.

**Основные возможности:**
- 📊 Мониторинг температуры — отображение текущих показателей датчиков
- ⚙️ Управление режимами — переключение между автоматическим и ручным управлением
- 📱 Адаптивный интерфейс — корректное отображение на всех устройствах
- 🐳 Docker-поддержка — быстрый запуск одной командой `docker-compose up`
- 🔒 Безопасность — базовая валидация запросов на стороне API

[🔗 Ссылка на репозиторий](https://github.com/Jekarobot/myheat-onboard)

### 🎓 MQ-Diploma

**Назначение:** Адаптивный лендинг, разработанный в рамках дипломного проекта курса «Мобильный веб-разработчик». Многостраничный сайт для демонстрации навыков вёрстки, адаптивного дизайна и работы с медиа-контентом. Основная цель — создание современного, отзывчивого интерфейса, корректно отображающегося на всех устройствах.

**Технологии и стек:** HTML5 (семантическая разметка), CSS3 (кастомные свойства и медиа-запросы), SVG (векторные иконки), Web-шрифты (WOFF, WOFF2, TTF — OpenSans), адаптивная вёрстка (поддержка десктопных, планшетных и мобильных разрешений).

**Архитектура и особенности:** Модульный принцип организации — отдельные папки для CSS, шрифтов, изображений и SVG-иконок. Адаптивность реализована через медиа-запросы и набор изображений для разных разрешений. Оптимизация через современные форматы шрифтов (WOFF2) и векторную графику. Семантическая вёрстка с тегами `<header>`, `<main>`, `<section>`, `<footer>` для улучшения SEO и доступности.

**Основные возможности:**
- Адаптивный дизайн — корректное отображение на экранах от 320px до 1920px
- Медиа-галерея — поддержка изображений в трёх разрешениях (десктоп, планшет, мобильный)
- Формы обратной связи — интерактивные элементы с SVG-иконками
- Трендовые блоки — секции с актуальными постами и товарами
- Кросс-браузерная совместимость — поддержка современных браузеров

### 🎯 DOM — Интерактивное веб-приложение для работы с DOM-деревом

**Назначение:** Учебное веб-приложение, демонстрирующее возможности манипуляции DOM-деревом с помощью JavaScript. Основная цель — показать, как динамически изменять содержимое страницы, обрабатывать события и визуализировать игровую логику. Приложение реализует простую механику кликов по появляющимся объектам.

**Технологии и стек:** JavaScript (ES6+), Webpack 5 (три конфигурации: общая, dev, prod), ESLint, HTML5 + CSS3, Yarn.

**Архитектура и особенности:** Модульная архитектура — вся логика вынесена в отдельный файл `src/js/app.js`, а основной скрипт `src/index.js` служит точкой входа. Стили организованы в едином CSS-файле. Webpack настроен для разработки и продакшена, что обеспечивает удобную разработку с hot-reload и оптимизированную сборку для деплоя. В проекте используются изображения (гоблин, молоток, логотип Netology) для визуализации игровых элементов.

**Основные возможности:**
- Динамическое создание DOM-элементов — объекты (гоблины) появляются в случайных ячейках сетки
- Обработка событий клика — при клике на гоблина срабатывает анимация и засчитывается попадание
- Визуальная обратная связь — использование изображений для игровых объектов и курсора
- Адаптивная верстка — стили обеспечивают корректное отображение на разных экранах
- Готовая инфраструктура — настроенный линтинг, сборка и управление зависимостями

### 🚆 Railway Booking — Сервис для покупки железнодорожных билетов

**Назначение:** Современное веб-приложение для поиска и бронирования железнодорожных билетов. Проект предоставляет интуитивно понятный интерфейс для выбора маршрутов, просмотра деталей поездов и оформления заказов в несколько шагов.

**Технологии и стек:** React 18, TypeScript, Vite, ESLint, Feature-Sliced Design (FSD).

**Архитектура и особенности:** Проект построен по методологии **Feature-Sliced Design**, что обеспечивает чёткое разделение на слои: `app`, `pages`, `features`, `entities`, `widgets`, `shared`. Используются контекстные провайдеры (`SearchProvider`, `OrderBuildProvider`, `PopupProvider`, `TrainDetailsProvider`) для управления глобальным состоянием. Данные обрабатываются через mock-данные и API-слой, а UI-компоненты вынесены в `shared/ui`.

**Основные возможности:**
- 🔍 Поиск билетов — гибкая форма с календарём и фильтрами
- 🚄 Просмотр поездов — карточки с деталями, слайдеры и анимации
- 🧾 Оформление заказа — пошаговый процесс с выбором мест, пассажиров и оплатой
- 📱 Адаптивный дизайн — поддержка мобильных устройств и десктопов
- 🎯 Дополнительные секции — «Как это работает», «Отзывы», «Последние билеты»

[🔗 Ссылка на репозиторий](https://github.com/Jekarobot/railway-booking)
