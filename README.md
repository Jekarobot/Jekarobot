### 📦 GitHub Repo Manager

**Назначение:** GitHub Repo Manager — это инструмент для автоматизации управления репозиториями GitHub и обновления профиля README. Проект позволяет генерировать структурированные списки проектов, создавать саммари на основе AI-анализа и автоматически синхронизировать данные между репозиториями и профилем пользователя.

**🛠️ Ключевые технологии и стек:** Проект построен на **TypeScript** с использованием **Node.js** для серверной части и CLI-интерфейса. В качестве AI-ассистента используется **DeepSeek API** для генерации саммари. Веб-интерфейс реализован на чистом **HTML/CSS/JavaScript** без фреймворков. Для контейнеризации применяется **Docker** с **docker-compose**.

**🏗️ Архитектура и особенности реализации:** Архитектура разделена на три основных модуля: **CLI** для командной строки, **сервер** с REST API и **сервисы** для работы с GitHub, DeepSeek и файловой системой. Особенностью является модульная структура с четким разделением ответственности — каждый сервис отвечает за свою задачу (работа с репозиториями, генерация README, санитизация данных). Конфигурация гибко настраивается через JSON-файлы.

**✨ Основные возможности:**
- 🔄 **Автоматическая синхронизация** репозиториев с профилем README
- 🤖 **AI-генерация саммари** проектов с помощью DeepSeek
- 📝 **Генерация структурированных списков** проектов в формате Markdown
- 🚀 **Push-уведомления** с подтверждением изменений
- 🧹 **Санитизация данных** для безопасной работы с пользовательским вводом
- 🐳 **Docker-поддержка** для простого развертывания
- ⚙️ **Гибкая конфигурация** через JSON-файлы и переменные окружения

🔗 [Перейти к проекту](https://github.com/Jekarobot/github-repo-manager)

### 🔔 Theia Extension Notifications

**Назначение:** Расширение для Eclipse Theia, реализующее систему уведомлений с поддержкой браузерных и серверных компонентов. Проект демонстрирует создание модульного расширения для Theia с разделением на frontend, backend и общие модули, а также включает контейнеризацию для развертывания. **Это мой дебют в создании расширений для новой для меня IDE.**

**🛠️ Ключевые технологии и стек:**
- **Eclipse Theia** — платформа для построения IDE
- **TypeScript** — основной язык разработки
- **Node.js** — серверная часть расширения
- **Docker** — контейнеризация приложения
- **Yarn** — менеджер зависимостей

**🏗️ Архитектура и особенности реализации:** Проект построен по модульной архитектуре с разделением на три слоя:
- **browser/** — клиентская часть для работы в браузере
- **common/** — общие интерфейсы и типы
- **node/** — серверная логика на Node.js

Расширение интегрируется в Theia через стандартные механизмы расширений (package.json с конфигурацией theiaExtension). Для сборки используется монорепозиторий с TypeScript и tsconfig.json, а Dockerfile в `theia-app/` позволяет развернуть полноценное приложение Theia с расширением.

**✨ Основные возможности:**
- Отображение уведомлений в интерфейсе Theia
- Поддержка различных типов оповещений (информационные, предупреждения, ошибки)
- Серверная обработка и отправка уведомлений
- Контейнеризация для простого развертывания
- Модульная архитектура для легкого расширения функционала

🔗 [Перейти к проекту](https://github.com/Jekarobot/theia-extension-notifications)

### 🏦 BHJ-Diploma — Система управления банковскими счетами

**Назначение:** Проект представляет собой веб-приложение для управления банковскими счетами и транзакциями. Система позволяет пользователям создавать счета, выполнять переводы средств и отслеживать историю операций через интуитивно понятный интерфейс. Решение разработано в рамках дипломного проекта и демонстрирует навыки построения полноценного клиент-серверного приложения.

**🛠️ Ключевые технологии и стек:**
- **Фронтенд:** JavaScript (ES6+), HTML5, CSS3, AdminLTE (шаблон интерфейса)
- **Бэкенд:** Node.js, Express.js (маршрутизация API)
- **Хранилище данных:** JSON-файл (`db.json`) в качестве базы данных
- **Архитектура:** Модульная структура с разделением на API, UI-компоненты и маршруты

**🏗️ Архитектура и особенности реализации:** Приложение построено по принципу одностраничного приложения (SPA) с серверной частью на Express. Фронтенд организован в виде модулей: `App.js` — главный контроллер, `api/` — слой взаимодействия с сервером, `ui/` — компоненты интерфейса. Серверная часть включает маршруты для работы с пользователями (`routes/user.js`), счетами (`routes/account.js`) и транзакциями (`routes/transaction.js`). Документация API и руководства по функциям хранятся в папке `md/`.

**✨ Основные возможности:**
- 🔐 **Регистрация и аутентификация** пользователей с хранением сессий
- 💳 **Создание и управление счетами** с отображением баланса
- 💸 **Выполнение транзакций** между счетами с валидацией данных
- 📊 **Просмотр истории операций** с фильтрацией и сортировкой
- 🎨 **Адаптивный интерфейс** на основе AdminLTE с боковым меню и модальными окнами
- 📝 **Асинхронные формы** для ввода данных без перезагрузки страницы

🔗 [Перейти к проекту](https://github.com/Jekarobot/bhj-diploma)

### 🔥 MyHeat-Onboard

**Назначение:** MyHeat-Onboard — это легковесное веб-приложение для управления и мониторинга системы отопления "MyHeat". Проект предоставляет удобный интерфейс для быстрого запуска и настройки оборудования, а также отображает ключевые параметры работы системы в реальном времени. **PHP я быстро освоил с 0 для этого задания.**

**🛠️ Технологический стек:**
- **Frontend:** HTML5, CSS3, JavaScript (Vanilla JS)
- **Backend:** PHP (API-слой)
- **Инфраструктура:** Docker, Docker Compose
- **Графика:** SVG (логотип), JPG/PNG (промо-изображения)

**🏗️ Архитектура и особенности:** Проект построен по принципу клиент-серверной архитектуры:
- **Frontend** (`index.html`, `index.css`, `index.js`) — одностраничное приложение (SPA) с динамическим обновлением данных.
- **Backend** (`api/index.php`) — REST API для обработки запросов и взаимодействия с системой отопления.
- **Контейнеризация** (`Dockerfile`, `docker-compose.yml`) — обеспечивает изолированное окружение и простой деплой.

Особенности реализации:
- Минималистичный дизайн с акцентом на читаемость данных.
- Асинхронная загрузка информации через Fetch API.
- Поддержка адаптивной верстки для мобильных устройств.

**✨ Основные возможности:**
- 📊 **Мониторинг температуры** — отображение текущих показателей датчиков.
- ⚙️ **Управление режимами** — переключение между автоматическим и ручным управлением.
- 📱 **Адаптивный интерфейс** — корректное отображение на всех устройствах.
- 🐳 **Docker-поддержка** — быстрый запуск одной командой `docker-compose up`.
- 🔒 **Безопасность** — базовая валидация запросов на стороне API.

### 🚆 Railway Booking — Сервис для покупки железнодорожных билетов

**Назначение:** Railway Booking — это современное веб-приложение для поиска и бронирования железнодорожных билетов. Проект предоставляет интуитивно понятный интерфейс для выбора маршрутов, просмотра деталей поездов и оформления заказов в несколько шагов. **Этот проект сделан руками — от первой строчки кода до финального дизайна.**

**🛠 Ключевые технологии и стек:**
- **React 18** + **TypeScript** — основа фронтенда
- **Vite** — быстрая сборка и разработка
- **ESLint** — поддержка качества кода
- **Feature-Sliced Design (FSD)** — архитектурная методология

**🏗 Архитектура и особенности реализации:** Проект построен по методологии **Feature-Sliced Design**, что обеспечивает чёткое разделение на слои: `app`, `pages`, `features`, `entities`, `widgets`, `shared`. Используются **контекстные провайдеры** (`SearchProvider`, `OrderBuildProvider`, `PopupProvider`, `TrainDetailsProvider`) для управления глобальным состоянием. Данные обрабатываются через **mock-данные** и API-слой, а UI-компоненты вынесены в `shared/ui`.

**✨ Основные возможности:**
- 🔍 **Поиск билетов** — гибкая форма с календарём и фильтрами
- 🚄 **Просмотр поездов** — карточки с деталями, слайдеры и анимации
- 🧾 **Оформление заказа** — пошаговый процесс с выбором мест, пассажиров и оплатой
- 📱 **Адаптивный дизайн** — поддержка мобильных устройств и десктопов
- 🎯 **Дополнительные секции** — «Как это работает», «Отзывы», «Последние билеты»

🔗 [Перейти к проекту](https://github.com/Jekarobot/railway-booking)
