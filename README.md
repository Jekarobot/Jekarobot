### 📂 github-repo-manager — Автоматизация управления репозиториями и профилем GitHub

**Назначение проекта** 🎯  
Это CLI-инструмент и веб-сервер для автоматизации обновления файла `PROJECTS.md` в вашем GitHub-профиле. Он собирает информацию о ваших публичных репозиториях, генерирует краткие AI-сводки (с помощью DeepSeek) и поддерживает актуальность списка проектов без ручного редактирования.

**Ключевые технологии и стек** 🛠️  
- **TypeScript** — основной язык разработки (строгая типизация, модульная архитектура).  
- **Node.js** — среда выполнения для CLI и сервера.  
- **Express.js** — веб-сервер с middleware для обработки push-уведомлений от GitHub.  
- **DeepSeek API** — генерация кратких описаний репозиториев.  
- **Docker** — контейнеризация для простого развертывания.

**Архитектура и особенности реализации** 🏗️  
Проект разделён на три слоя:  
1. **CLI** (`src/cli/`) — точка входа для ручного запуска.  
2. **Core** (`src/core/`) — конфигурация, логирование, типы.  
3. **Services** (`src/services/`) — бизнес-логика: работа с GitHub API, генерация сводок, обновление README/PROJECTS.md.  
Сервер (`src/server/`) поддерживает webhook для автоматического обновления при push-событиях. Конфигурация через `repos.config.json` позволяет гибко настраивать фильтры репозиториев.

**Основные возможности** ✨  
- 🔄 **Автоматическое обновление** `PROJECTS.md` при каждом push в репозиторий.  
- 🤖 **AI-сводки** для каждого проекта (через DeepSeek).  
- 🧹 **Санитизация** описаний (удаление лишних символов, форматирование).  
- 🐳 **Готов к Docker** — `Dockerfile` и `docker-compose.yml` для быстрого запуска.  
- 📝 **Пример конфигурации** (`repos.config.example.json`) для быстрого старта.

🔗 [github.com/Jekarobot/github-repo-manager](https://github.com/Jekarobot/github-repo-manager)

### 📬 theia-extension-notifications — Система уведомлений для Eclipse Theia

**Назначение проекта** 🎯  
Расширение для Eclipse Theia, реализующее систему уведомлений с возможностью настройки и визуализации. Проект предоставляет интеграцию уведомлений в среду разработки через WebView-интерфейс, позволяя разработчикам получать и управлять оповещениями непосредственно в IDE. Решение ориентировано на улучшение пользовательского опыта при работе с Theia-приложениями.

**Ключевые технологии и стек** 🛠️  
- **Язык**: TypeScript (Node.js)  
- **Платформа**: Eclipse Theia (расширение)  
- **Инструменты**: Yarn, Jest (тестирование), Docker (контейнеризация)  
- **Архитектура**: WebView-провайдер для фронтенда, серверная логика на Node.js

**Архитектура и особенности реализации** 🏗️  
Проект построен по модульной схеме: основное расширение (`theia-notification-extension`) содержит логику обработки уведомлений, хранилище (`storage.ts`) и WebView-провайдер (`webview-provider.ts`) для отображения интерфейса. Тестирование выполняется с моком VSCode API (`vscode-mock.js`). Для развёртывания предусмотрен Docker-образ в `theia-app/Dockerfile`.

**Основные возможности** ✨  
- 📌 Создание и отображение уведомлений в WebView-панели  
- 🔧 Настройка типов уведомлений и их параметров  
- 💾 Персистентное хранение истории уведомлений  
- 🧪 Полное покрытие модульными тестами (Jest)  
- 🐳 Готовая конфигурация для Docker-развёртывания

🔗 [github.com/Jekarobot/theia-extension-notifications](https://github.com/Jekarobot/theia-extension-notifications)

### 🚀 AsyncCheck — Асинхронная проверка данных

**Назначение проекта** 🎯  
AsyncCheck — это веб-приложение для асинхронной проверки и мониторинга состояния удалённых задач или сервисов. Проект позволяет запускать длительные операции в фоне, отслеживать их прогресс и получать уведомления о завершении без блокировки пользовательского интерфейса.

**Ключевые технологии и стек** 🛠️  
- **Backend**: NestJS (TypeScript), Docker, PostgreSQL (через Docker Compose), ESLint  
- **Frontend**: React (TypeScript), Vite, компонентный подход с разделением на features  
- **Инфраструктура**: Docker Compose для оркестрации, модульное тестирование (Jest) и e2e-тесты

**Архитектура и особенности реализации** 🏗️  
Проект построен по модульной архитектуре: backend на NestJS использует модули `common` (общие утилиты) и `jobs` (управление задачами), а frontend разделён на `api`, `app`, `components` и `features` для чёткого разделения ответственности. Асинхронность реализована через очередь задач с возможностью отслеживания статуса через WebSocket или polling. Docker-контейнеры обеспечивают изолированное окружение и простой деплой.

**Основные возможности** ✨  
- Запуск асинхронных задач с уникальным идентификатором  
- Мониторинг прогресса выполнения в реальном времени  
- REST API для управления задачами (создание, отмена, получение статуса)  
- Гибкая конфигурация через переменные окружения  
- Масштабируемая архитектура для добавления новых типов проверок

🔗 [github.com/Jekarobot/AsyncCheck](https://github.com/Jekarobot/AsyncCheck)

### 🚂 Railway Booking — Поиск и бронирование железнодорожных билетов

**Назначение проекта** 🎯  
Railway Booking — это современное веб-приложение для поиска и бронирования железнодорожных билетов. Проект предоставляет пользователям интуитивно понятный интерфейс для выбора маршрутов, просмотра детальной информации о поездах и оформления заказов. Решение ориентировано на создание бесшовного пользовательского опыта при планировании поездок.

**Ключевые технологии и стек** 🛠️  
- **Фреймворк**: React 18 с TypeScript для типизированной разработки  
- **Сборка**: Vite для быстрой разработки и оптимизированной сборки  
- **Стилизация**: CSS-модули для изолированных стилей компонентов  
- **Линтер**: ESLint с конфигурацией для React и TypeScript  
- **Архитектура**: Feature-Sliced Design (FSD) для масштабируемости

**Архитектура и особенности реализации** 🏗️  
Проект построен на архитектуре **Feature-Sliced Design**, что обеспечивает четкое разделение ответственности между слоями: `app` (конфигурация приложения), `pages` (страницы), `features` (функциональные модули), `widgets` (композитные компоненты), `entities` (бизнес-сущности) и `shared` (переиспользуемые утилиты). Используется **контекстное управление состоянием** через провайдеры (`SearchProvider`, `OrderBuildProvider`, `PopupProvider`, `TrainDetailsProvider`), что позволяет эффективно управлять данными без внешних библиотек состояния. Моковые данные (`mockData`) имитируют работу API для демонстрации функциональности.

**Основные возможности** ✨  
- **🔍 Поиск билетов**: Интерактивная форма с кастомным календарем и фильтрами для точного подбора маршрута  
- **🎫 Просмотр результатов**: Детальная информация о поездах с возможностью сортировки и фильтрации по цене, времени и типу вагона  
- **🛒 Оформление заказа**: Многошаговый процесс с выбором мест, вводом данных пассажиров и биллингом  
- **📱 Адаптивный интерфейс**: Слайдеры, попапы и интерактивные элементы для комфортной работы на любых устройствах  
- **⚡ Быстрая навигация**: Скролл к якорям и плавные переходы между страницами Home, Results и Success

🔗 [github.com/Jekarobot/railway-booking](https://github.com/Jekarobot/railway-booking)
