### 🚀 GitHub Repo Manager

**Назначение:**  
Умный инструмент для автоматизации управления репозиториями и генерации динамического профиля на GitHub. Проект анализирует ваши репозитории, создает структурированные списки проектов и автоматически обновляет файл `README.md` в профиле, используя ИИ для генерации кратких описаний.

**🛠️ Ключевые технологии и стек:**  
- **Backend:** Node.js, TypeScript  
- **CLI:** Commander.js  
- **Сервер:** Express.js  
- **ИИ-интеграция:** DeepSeek API  
- **Инфраструктура:** Docker, Docker Compose  

**🏗️ Архитектура и особенности реализации:**  
Проект построен по модульной архитектуре с четким разделением на CLI, серверную часть и сервисы. CLI-интерфейс позволяет запускать задачи вручную, а серверный модуль с middleware `push-confirm` автоматически реагирует на push-события в репозиториях. Ключевая особенность — использование DeepSeek API для генерации сжатых, но информативных описаний проектов, что экономит время и улучшает качество контента.

**✨ Основные возможности:**  
- **Автоматическая генерация списка проектов** в формате Markdown на основе конфигурации `repos.config.json`.  
- **Интеграция с DeepSeek AI** для создания кратких, осмысленных описаний каждого репозитория.  
- **Автоматическое обновление `README.md` профиля** при каждом пуше (через webhook или CLI).  
- **Гибкая настройка через конфигурационные файлы** (`PROJECTS.md`, `PROJECTS-EXAMPLE.md`).  
- **Контейнеризация** с помощью Docker для простого развертывания и масштабирования.

[🔗 GitHub Repo Manager](https://github.com/Jekarobot/github-repo-manager)

### 🔔 Theia Extension Notifications

**Назначение проекта**  
Расширение для Eclipse Theia, добавляющее систему уведомлений с поддержкой real-time оповещений. Проект позволяет разработчикам интегрировать кастомные уведомления в IDE на базе Theia, включая push-уведомления от внешних сервисов и внутриприложенные события. Решение ориентировано на улучшение пользовательского опыта и автоматизацию рабочих процессов.

**🚀 Ключевые технологии и стек**  
- **Фреймворк**: Eclipse Theia (TypeScript)  
- **Архитектура**: Модульная структура с разделением на browser/common/node  
- **Инструменты**: Yarn workspaces, TypeScript, Docker  
- **Инфраструктура**: Контейнеризация через Docker, монорепозиторий с `theia-app` и `theia-notification-extension`

**🏗 Архитектура и особенности реализации**  
Проект построен по принципу расширения Theia с тремя слоями:  
- **browser** — клиентская часть для отображения уведомлений в UI  
- **common** — общие типы и интерфейсы для передачи данных  
- **node** — серверная логика обработки и отправки уведомлений  

Особенности:  
- Использование `yarn.lock` для фиксации зависимостей  
- Документация в `docs/` с детальным описанием архитектуры, бэкенда и фронтенда  
- Dockerfile для развертывания Theia-приложения с расширением  

**✨ Основные возможности**  
- 📥 Прием и отображение уведомлений от внешних API  
- 🔄 Real-time обновления через WebSocket  
- 🎨 Кастомизация стилей и типов уведомлений  
- 🗂 Поддержка группировки и фильтрации  
- 🐳 Готовность к контейнеризации для CI/CD

[🔗 Theia Extension Notifications](https://github.com/Jekarobot/theia-extension-notifications)

### 🔥 MyHeat-Onboard

## 📋 Назначение проекта

**MyHeat-Onboard** — это легковесное одностраничное приложение для предварительной регистрации и онбординга пользователей в системе управления отоплением. Проект предоставляет простой и интуитивный интерфейс для сбора контактных данных и демонстрации возможностей сервиса. Решение ориентировано на быстрый запуск и минимальную конфигурацию.

## 🛠️ Ключевые технологии и стек

- **Frontend**: HTML5, CSS3, JavaScript (ванильный) — без фреймворков для максимальной производительности
- **Backend**: PHP 8+ (API-слой в `api/index.php`)
- **Инфраструктура**: Docker + Docker Compose для контейнеризации
- **Ассеты**: SVG-логотип, промо-изображения (JPEG/PNG)

## 🏗️ Архитектура и особенности реализации

Проект построен по принципу SPA (Single Page Application) с минимальным серверным API. Фронтенд полностью статический, все ассеты и стили хранятся локально. Бэкенд реализован через единственный PHP-скрипт, обрабатывающий POST-запросы. Контейнеризация через Docker обеспечивает изолированное окружение и простой деплой. В проекте предусмотрены файлы для документирования потенциальных проблем (`POTENTIAL_PROBLEMS.md`) и списка задач (`TASKS.md`).

## ✨ Основные возможности

- 📝 Форма онбординга с валидацией данных на клиенте и сервере
- 🎨 Адаптивный дизайн с фирменным стилем (белый логотип, промо-материалы)
- 🐳 Готовая Docker-среда для разработки и продакшена (`docker-compose.yml`)
- 📄 Чистая архитектура: все файлы структурированы по назначению (API, ассеты, конфигурация)
- 🔒 Безопасная обработка данных через PHP-бэкенд

Проект идеально подходит как стартовый шаблон для быстрого прототипирования сервисов онбординга.

[🔗 MyHeat-Onboard](https://github.com/Jekarobot/myheat-onboard)

### 🚆 Railway Booking — Система бронирования железнодорожных билетов

**Railway Booking** — это современное веб-приложение для поиска и покупки железнодорожных билетов, разработанное с использованием React и TypeScript. Проект предоставляет удобный интерфейс для выбора маршрутов, просмотра расписания и оформления заказов, имитируя реальный процесс бронирования.

## 🛠 Ключевые технологии и стек

- **React 18** с **TypeScript** для типизированной и масштабируемой разработки
- **Vite** для быстрой сборки и горячей перезагрузки модулей
- **ESLint** с конфигурацией для поддержания качества кода
- **Feature-Sliced Design (FSD)** — архитектурная методология для организации кода

## 🏗 Архитектура и особенности реализации

Проект построен по методологии **Feature-Sliced Design**, что обеспечивает чёткое разделение ответственности между слоями: `entities`, `features`, `widgets`, `pages` и `shared`. Контекстные провайдеры (`SearchProvider`, `OrderBuildProvider`, `PopupProvider`, `TrainDetailsProvider`) управляют глобальным состоянием приложения, а моковые данные (`mockData`) позволяют тестировать функциональность без реального API.

## ✨ Основные возможности

- 🔍 **Поиск билетов** с фильтрацией по дате, времени и типу поезда
- 🗓 **Интерактивный календарь** для выбора дат отправления и возвращения
- 🚃 **Детальный просмотр** информации о поездах, вагонах и местах
- 🛒 **Оформление заказа** с вводом данных пассажиров и выбором способа оплаты
- 📋 **История бронирований** и подтверждение успешной покупки
- 📱 **Адаптивный дизайн** для комфортного использования на мобильных устройствах
